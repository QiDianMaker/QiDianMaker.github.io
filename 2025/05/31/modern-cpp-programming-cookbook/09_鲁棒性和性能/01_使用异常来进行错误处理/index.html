使用异常来进行错误处理

异常是对程序运行时可能出现的异常情况的响应。它们允许将控制流转移到程序的另一部分。异常是一种更简单、更健壮的错误处理机制，而不是返回错误代码，这会使代码变得非常复杂和混乱。在本条款中，我们将研究与抛出和处理异常相关的一些关键方面。

准备工作

本条款要求对抛出异常（使用 throw 语句）和捕获异常（使用 try…catch 块）的基本制有所了解。本条款关注的是围绕异常的良好实践，而不是 C++ 语言中异常机制的细节。

原理分析

大多数函数必须表明其执行的成功或失败。这可以通过不同的方式实现。以下是几种可能性：

  • 返回一个错误代码（带有 success 的特殊值）来指示失败的具体原因：

  • 它的一个变体是返回一个布尔值，只表示成功或失败：

  • 另一种方法是返回无效对象、空指针或空 std::optional<T> 对象：

在任何情况下，都应该检查函数的返回值。这可能导致复杂、混乱、难以阅读和维护真实世界的代码。此外，无论函数是否成功，检查函数返回值的过程始终执行。另一方面，只有当函数失败时才会抛出和处理异常，这种情况应该比成功执行更少发生。这实际上可以产生比返回和测试错误代码更快的代码。

异常和错误代码并不相互排斥。异常只能用于在异常情况下传输控制流，而不能用于控制程序中的数据流。

类构造函数是不返回任何值的特殊函数。它们应该构造一个对象，但在失败的情况下，它们将无法用返回值指示这一点。异常应该是构造函数用来指示失败的机制。与资源获取即初始化 (RAII) 惯用法语一起，这确保了在所有情况下资源的安全获取和释放。另一方面，不允许异常离开析构函数。当发生这种情况时，程序调用std::terminate()异常终止。这是由于发生另一个异常而在堆栈展开期间调用析构函数的情况。当发生异常时，将从抛出异常的点到处理异常的块展开堆栈。这个过程包括销毁所有这些堆栈帧中的所有局部对象。

如果在此过程中被销毁的对象的析构函数抛出异常，则应该开始另一个堆栈展开进程，该进程与正在进行的堆栈展开进程相冲突。因此，程序异常终止。

在构造函数和析构函数中处理异常的经验法则如下：

  1. 使用异常指示构造函数中发生的错误。
  2. 不要抛出或让异常离开析构函数。

可以抛出任何类型的异常。但是，在大多数情况下，应该抛出临时变量并通过常量引用捕获异常。下面是抛出异常的一些准则：

  • 更倾向于抛出标准异常或从 std::exception 或其他标准异常派生的自己的异常。这样做的原因是标准库提供了用于表示异常的首选异常类。您应该使用已经可用的工具，当这些工具不够好时，可以根据标准工具构建自己的工具。这样做的主要好处是一致性和帮助用户通过基类 std::exception 捕获异常。

  • 避免抛出内置类型（如整数）的异常。这样做的原因是数字向用户传递的信息很少，用户必须知道它代表什么，而对象可以提供上下文信息。例如，语句throw 42; 不告诉用户任何信息，但抛出 access_denied_exception{};仅从类名就携带了更多的隐式信息，并且在数据成员的帮助下，它携带了有关异常情况的任何有用或必要的信息。

  • 当使用提供自己异常层次结构的库或框架时，最好从这个层次结构抛出异常，或者从它派生出自己的异常，至少在与它紧密相关的代码部分抛出异常。这样做的主要原因是为了保持使用库 API 的代码的一致性。

补充

如前一节所述，当您需要创建自己的异常类型时，请从可用的标准异常中派生它们，除非您正在使用具有自己的异常层次结构的库或框架。C++ 标准定义了几类需要考虑的异常：

  • logic_error 表示一个异常，该异常指示程序逻辑中的错误，例如无效参数、超出范围边界的索引等等。有各种标准派生类，如std::invalid_argument、std::out_of_range 和 std::length_error。

  • runtime_error 表示一个异常，它表示一个超出程序范围的错误，或者由于各种因素（包括外部因素，如溢出和下溢或操作系统错误）而无法预测的错误。C++ 标准还提供了几个来自 std::runtime_error 的派生类，包括 C++20 中的 std::overflow_error、std::underflow_error、std::system_error 和 std::format_error。

  • 以 bad_ 为前缀的异常，例如 std::bad_alloc、std::bad_cast 和std::bad_function_call，表示程序中的各种错误，例如分配内存失败、动态强制转换或进行函数调用失败，等等。

所有这些异常的基类都是 std::exception。它有一个名为 what() 的非抛出虚方法，该方法返回一个指向表示错误描述的字符数组的指针。

当您需要从标准异常派生自定义异常时，请使用适当的类别，例如逻辑或运行时错误。如果这些类别都不合适，那么可以直接从 std::exception 派生。以下是可以用来从标准异常派生的可能解决方案的列表：
