尽可能使用 auto

自动类型推导是现代 C++ 中最重要、使用最广泛的特性之一。新的 C++ 标准使得在各种上下文中使用 auto 作为类型的占位符并让编译器推断出实际类型成为可能。在 C++11 中，auto 可用于声明局部变量和声明具有尾随返回类型的函数的返回类型。在 C++14 中，auto 可用于函数的返回类型，而无需指定尾随类型，也可用于 lambda 表达式中的参数声明。未来的标准版本可能会将 auto 的使用扩展到更多的情况。在这些情况下使用 auto 有几个重要的好处，我们将在 原理分析 部分详细讨论。开发人员应该意识到这一点，并尽可能选择 auto。Andrei Alexandrescu 还为此专门创造了一个术语，并由 Herb Sutter 发扬光大——几乎总是 auto（almost always auto, AAA）。

原理分析

auto 说明符基本上是实际类型的占位符。当使用 auto 时，编译器从以下实例推断出实际类型：

  • 当 auto 用于声明变量时，从初始化变量的表达式的类型中获取。
  • 当 auto 用作函数返回类型的占位符时，从尾随的返回类型或函数返回表达式的类型中获取。

在某些情况下，有必要指定成特定的类型。例如，在上一节的第一个示例中，编译器将 s 的类型推导为 char const *。如果目的是使用 std::string，则必须显式指定类型。类似地，v 的类型推导为 std::initializer_list<int>。然而，目的可能是一个std::vector<int>。在这种情况下，必须在赋值的右侧显式指定类型。

使用 auto 说明符而不是实际类型有一些重要的好处：以下是一些可能是最重要的因素：

  • 不可能不初始化变量。这是开发人员在声明指定实际类型的变量时常犯的错误。但是，这在 auto 中是不可能的，它需要对变量进行初始化才能推断类型。

  • 使用 auto 可确保始终使用正确的类型，并且不会发生隐式转换。考虑下面的例子，我们将 vector 的大小保存到一个局部变量。在第一种情况下，变量的类型是 int，尽管 size() 方法返回值的类型是 size_t。这意味着将发生从 size_t 到 int 的隐式类型转换。然而，对类型使用 auto 将推断出正确的类型；也就是 size_t：

  • 使用 auto 促进了良好的面向对象实践，比如更倾向于接口而不是实现。指定的类型数量越少，代码就越泛型，对将来的更改也越开放，这也是面向对象编程的基本原则。

  • 这意味着更少的输入，更少地关注我们不关心的实际类型。通常情况下，即使我们显式地指定了类型，我们实际上也并不关心它。一个非常常见的例子是迭代器，但还有更多。当你想在一个范围内迭代时，你不关心迭代器的实际类型。你只对迭代器本身感兴趣；因此，使用 auto 可以节省键入可能很长的名称的时间，并帮助你专注于实际代码而不是类型名称。在下面的例子中，在第一个 for 循环中，我们显式使用了迭代器的类型。有很多文本要输入；长语句实际上会降低代码的可读性，而且你还需要知道实际上并不关心的类型名称。第二个带有 auto 说明符的循环看起来更简单，而且使你不必输入和关心实际类型：

  • 使用 auto 声明变量提供了一致的编码风格，类型总是在右侧。如果动态分配对象，则需要在赋值语句的左右两边都写上类型，例如 int* p = new int(42)。使用 auto，类型只在右侧指定一次。

  • 对于多词类型，如 long long、long double 或 struct foo，不能使用 auto。然而，在第一种情况下，可能的解决方法是使用字面量或类型别名；至于第二种，为了与 C 兼容，在 C++ 中只支持以这种形式使用 struct/class，无论如何都应该避免：

  • 如果您使用 auto 说明符，但仍然需要知道类型，那么在大多数 IDE 中都可以这样做，例如，将光标放在变量上。然而，如果你离开了 IDE，这已经不可能了，知道实际类型的唯一方法是自己从初始化表达式中推断出来，这可能意味着要在代码中搜索函数返回类型。
