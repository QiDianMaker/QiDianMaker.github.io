创建类型别名和别名模板

在 C++ 中，可以创建用来代替类型名称的同义词。这可以通过创建一个 typedef 声明来实现。这在很多情况下都很有用，比如为类型或函数指针创建更短或更有意义的名称。然而，typedef 声明不能与模板一起用于创建模板类型别名。例如，std::vector<T> 不是一种类型（std::vector<int> 是一种类型），而是一种所有类型的家族，其中的类型占位符 T 可以被替换为实际类型。

在 C++11 中，类型别名是另一个已经声明的类型的名称，别名模板是另一个已经声明的模板的名称。这两种类型的别名都是通过新的 using 语法引入的。

原理分析

typedef 声明为类型引入了同义词（换句话说就是别名）。它不引入另一种类型（如类、结构、联合或枚举声明）。通过typedef 声明引入的类型名遵循与标识符名称相同的隐藏规则。它们也可以重新声明，但只能引用相同的类型（因此，您可以拥有有效的多个 typedef 声明，在翻译单元中引入相同的类型名称同义词，只要它是同一类型的同义词）。下面是 typedef 声明的典型例子：

类型别名声明等同于类型定义声明。它可以出现在块作用域、类作用域或名称空间作用域中。根据 C++11 第 7.1.3.2 段：

A typedef-name can also be introduced by an alias declaration. The identifier following the using keyword becomes a typedef-name and the optional attributespecifier-seq following the identifier appertains to that typedef-name. It has the same semantics as if it were introduced by the typedef specifier. In particular, it does not define a new type and it shall not appear in the type-id.

但是，在为数组类型和函数指针类型创建别名时，别名声明对别名的实际类型更具可读性和清晰度。在使用方法小节的例子中，很容易理解的是，array_t 是包含 10 个整数的数组类型的名称，而 fn 是一个函数类型的名称，该函数类型接受byte 和 double 类型的两个形参并返回 void。这也与声明 std::function 对象的语法一致（例如，std::function<void(byte, double)> f）。

重要的是要注意以下几点：

  • 别名模板不能部分或显式特化。
  • 在推导模板形参时，绝不会通过模板实参推导推导出别名模板。
  • 特化别名模板时产生的类型不允许直接或间接地使用它自己的类型。

新语法的加入的目的是定义别名模板。这些模板在特化时等同于用别名模板的模板实参替换 type-id 中的模板形参的结果。
